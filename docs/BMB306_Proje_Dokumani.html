1 – Proje Tanımı ve Ekip Bilgisi

1.1 Proje Özeti

CloudFile, kullanıcıların dosyalarını bulut ortamında depolayıp yönetebileceği, başkaları ile paylaşabileceği ve farklı rol seviyelerinde birlikte çalışabileceği bir web uygulamasıdır. Uygulama; ASP.NET Core Web API (.NET 10) tabanlı RESTful bir backend ile React 18 + Vite tabanlı modern bir frontend'den oluşmaktadır. Veritabanı olarak PostgreSQL, kimlik doğrulama için JWT ve şifre hashleme için BCrypt kullanılmıştır.

1.2 Temel Özellikler

  • JWT tabanlı kullanıcı kayıt / giriş sistemi
  • Üç katmanlı rol sistemi: Admin, Manager, User
  • İç içe (hiyerarşik) klasör yönetimi
  • Dosya yükleme, indirme, soft-delete ile silme
  • Otomatik dosya versiyonlama (aynı isimde dosya tekrar yüklendiğinde yeni versiyon oluşturulur)
  • Token tabanlı paylaşılabilir (anonim erişimli) link oluşturma
  • Kullanıcı başına depolama kotası (varsayılan 1 GB, admin tarafından değiştirilebilir)
  • Admin paneli: kullanıcı yönetimi, kota/rol değiştirme, istatistikler
  • Token tabanlı şifre sıfırlama

→

→
🗄️ PostgreSQL
1Kullanıcı Login formuna e-posta ve şifre girer, "Giriş Yap" düğmesine basar.
2React, POST /api/auth/login isteği gönderir (JSON body: {email, password}).

7Şifre yanlışsa → 401 Unauthorized döner.
8JwtHelper.GenerateToken() ile Claims içeren JWT oluşturulur (sub, email, role, exp).
9Controller, 200 OK ile {token, user} döner.
10React, token'ı localStorage'a kaydeder; AuthContext güncellenir.
11Role'e göre /admin veya / (Dashboard) sayfasına yönlendirilir.

9 – Teknik Rapor ve API Dokümantasyonu

9.1 Mimari Genel Bakış

Uygulama, standart katmanlı mimari yaklaşımını benimser: Presentation Layer (React SPA) → API Layer (ASP.NET Core Controllers) → Service Layer (iş mantığı) → Data Layer (EF Core + PostgreSQL).

  • Frontend: React 18 + Vite (port 3000). Axios interceptor aracılığıyla tüm isteklere Authorization: Bearer <token> header'ı otomatik eklenir. React Router DOM v6 ile sayfa yönlendirmesi yapılır.
  • Backend: ASP.NET Core Web API (.NET 10), port 5200. CORS politikası React origin'ini kabul edecek şekilde yapılandırılmıştır.
  • Kimlik Doğrulama: Giriş sonrası 7 günlük JWT token üretilir. Token payload: sub (userId), email, role, exp.
  • Depolama: Dosyalar uploads/ klasörüne GUID-tabanlı isimlendirme ile kaydedilir. Versiyonlar birbirinden bağımsız fiziksel dosyalardır.

10 – Kullanım Kılavuzu

10.1 Kurulum ve Başlatma

Önkoşullar

  • .NET 10 SDK (dotnet.microsoft.com adresinden indirilebilir)
  • PostgreSQL 15+ (varsayılan port 5432)
  • Node.js 18+ ve npm

Backend Kurulumu

  1. Proje kök dizinine gidin: cd backend/CloudFile.Api
  2. appsettings.json dosyasında bağlantı dizesini düzenleyin:
    "DefaultConnection": "Host=localhost;Port=5432;Database=cloud_file_db;Username=postgres;Password=SIFRE"
  3. Veritabanı migration'larını uygulayın: dotnet ef database update
  4. API sunucusunu başlatın: dotnet run
  5. API: http://localhost:5200  |  Swagger: http://localhost:5200/swagger

Frontend Kurulumu

  1. Frontend dizinine gidin: cd frontend/cloud-file-ui
  2. Bağımlılıkları yükleyin: npm install
  3. Geliştirme sunucusunu başlatın: npm run dev
  4. Uygulama: http://localhost:3000

10.2 Kullanıcı Arayüzü Akışı

Kayıt ve Giriş

  1. Tarayıcıda http://localhost:3000 adresine gidin.
  2. "Kayıt Ol" linkine tıklayarak hesap oluşturun (ad, e-posta, şifre).
  3. Giriş sayfasında e-posta ve şifrenizi girerek sisteme erişin.
  4. Admin hesabıyla giriş yapıldığında /admin paneli açılır.

Dashboard – Dosya ve Klasör Yönetimi

  1. Klasör Oluşturma: "📁 Klasör" düğmesine basın, ad girin, Enter'a basın veya "Oluştur"a tıklayın.
  2. Klasöre Girme: Klasör ikonuna çift tıklayın veya "→" düğmesine basın. Üst kısımdaki breadcrumb ile geri dönebilirsiniz.
  3. Dosya Yükleme: "⬆️ Yükle" düğmesine tıklayın; açılan modal'da dosya seçin ve "Yükle"ye basın. Aynı isimde dosya mevcutsa yeni versiyon oluşturulur.
  4. Dosya Arama: Üst kısımdaki arama kutusuna dosya adını yazmaya başlayın; liste anlık filtrelenir.
  5. Dosya İndirme: Dosya satırının üzerine geldiğinizde görünen "⬇️" düğmesine tıklayın.
  6. Paylaşım Linki: "🔗" düğmesine tıklayın; link otomatik olarak panoya kopyalanır. Bu link, hesap sahibi olmayan kişilerin de dosyayı indirmesine olanak tanır.
  7. Dosya Silme: "🗑️" düğmesine tıklayın ve onay verin. Dosya soft-delete ile işaretlenir (fiziksel dosya silinmez).

Şifre Sıfırlama

  1. Giriş sayfasında "Şifremi unuttum" linkine tıklayın.
  2. E-posta adresinizi girin ve "Gönder"e basın.
  3. Ekranda görünen reset token'ını kopyalayın (gerçek sistemde bu e-posta ile gelir).
  4. "Şifreyi sıfırlamak için tıkla" linkine basın veya /reset-password?token=TOKEN adresine gidin.
  5. Yeni şifrenizi girin ve "Şifremi Sıfırla"ya basın.
  6. 2 saniye sonra otomatik olarak giriş sayfasına yönlendirilirsiniz.

Admin Paneli

  1. Admin rolüyle giriş yaptığınızda otomatik olarak /admin sayfasına yönlendirilirsiniz.
  2. Kullanıcı listesinden herhangi bir kullanıcının kotasını (GB cinsinden) veya rolünü değiştirebilirsiniz.
  3. Üst kısımdaki istatistik kartlarında toplam kullanıcı sayısı, toplam dosya sayısı ve toplam kullanılan depolama alanı görüntülenir.
  4. Hesap aktiflik durumunu değiştirerek kullanıcıları geçici olarak devre dışı bırakabilirsiniz.

10.3 Bilinen Kısıtlamalar ve Notlar

  • Şifre sıfırlama token'ı demo amaçlı API response'da dönmektedir; üretim ortamında e-posta servisi (SMTP / SendGrid) entegre edilmelidir.
  • Tek dosya yükleme limiti 100 MB olarak ayarlanmıştır (appsettings.json → MaxFileSizeBytes).
  • Dosyalar soft-delete ile silinir; fiziksel dosya disk üzerinde kalmaya devam eder. Disk temizliği için ayrı bir background job eklenmelidir.
  • Mevcut depolama backend'i sunucu diski kullanmaktadır; yüksek ölçekli kullanım için MinIO veya AWS S3 entegrasyonu önerilir.
  • JWT secret key üretim ortamında güvenli bir key-vault veya environment variable aracılığıyla yönetilmelidir.
